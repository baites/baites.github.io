# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What this is

A Jekyll static blog ("Notes of an interdisciplinarian") published via GitHub Pages at `baites.github.io`. Content is math/science/engineering long-form posts with heavy LaTeX and pseudocode. Theme is `minima`; Markdown engine is `kramdown`.

## Development workflow (Docker)

Development happens inside a long-running container named `jekyll-blog` (built from `jekyll.dockerfile`, an Arch Linux image with Ruby/Bundler). The repo is bind-mounted into the container. On this Windows host the container is already created and started.

Typical loop:

```bash
docker exec -it jekyll-blog /bin/bash        # enter the container
bundle exec jekyll serve --future --host 0.0.0.0 --force_polling --livereload
# then open http://localhost:4000/
```

- `--future` is required to see most in-progress content — see "Drafts" below.
- `--force_polling --livereload` are needed because file-change events don't propagate reliably across the Windows bind mount.
- `bundle update` (inside the container) refreshes gem dependencies; it rewrites `Gemfile.lock`.
- Editing `_config.yml` does **not** hot-reload — restart `jekyll serve` after changing it.
- `docker build -t jekyll -f jekyll.dockerfile .` rebuilds the image if the container needs recreating (see README for the `docker create ... --name jekyll-blog` invocation). Note the README references a `Dockerfile`; the actual file is `jekyll.dockerfile`.

There is no test suite or linter — verification is visual, by serving the site and viewing the rendered page.

## Drafts convention

Posts use their filename date as the publish date. Anything dated in the future (e.g. `2029-*`, `3000-*`) is an unpublished draft and is only rendered when `jekyll serve` is run with `--future`. `future: false` in `_config.yml` keeps these hidden on the live GitHub Pages site. To "publish" a draft, rename it to a past/current date.

## Post authoring conventions

Front matter drives per-post feature toggles. Example:

```yaml
---
layout: post
title: A pair of pitches with a twist
date: 2025-12-10 18:00:00 -0400
author: Victor E. Bazterra
categories: music math music-notes-series
javascript:
  katex: true          # loads KaTeX assets + auto-render (see _includes/head.html)
---
```

- `javascript.katex: true` loads KaTeX CSS/JS. Other supported toggles wired in `_includes/head.html`: `javascript.mathjax` (`true` for local, `'cdnjs'` for CDN) and `javascript.pseudocode`.
- **Categories** are space-separated. Any category whose name contains `series` (e.g. `music-notes-series`) causes the post to be listed on `series.md`; all categories are listed on `categories.md`. Both pages are generated by looping `site.categories`.

### Math delimiters (KaTeX auto-render)

Configured in `_includes/head.html`. Inline: `$...$`. Display: `%%...%%` (custom), `\[...\]`, or `\(...\)`.

Because kramdown processes Markdown before KaTeX runs, LaTeX subscripts and pipes inside `$...$` are escaped in existing posts as `\_` and `\|` (e.g. `f\_1`, `\mathbf{e}\_1`) to stop kramdown from eating `_` as emphasis. Match this style when editing math or the render will break.

### Custom includes

- `{% include image file="foo.svg" scale="80%" align="center" %}` — renders `/assets/images/foo.svg`. `scale`/`align` optional.
- `{% include statement/theorem name="..." %}` … content … `{% include statement/end %}` — boxed math statements. Types live in `_includes/statement/`: `theorem`, `lemma`, `corollary`, `proposition`, `definition`, `example`, `remark`, `problem`. Each opens a `<div>`; `statement/end` closes it. `statement/disclaimer` is self-closing (no `end` needed). Styled by `assets/css/statement.css`.
- `{% include katex id="..." code="..." displayMode=true %}` and `{% include pseudocode id="..." code="..." %}` — explicit render into a target element (used when auto-render isn't enough).

### Figures

Diagrams are authored in draw.io as `assets/images/*.drawio` and exported to `.svg` (same basename) which is what posts reference via the `image` include. The `.$*.drawio.bkp` files are draw.io autosave backups and should not be committed.

## Layout / structure notes

- `_layouts/`: `post.html` and `home.html` both extend the theme's `default`; they are local overrides of minima. `post.html` adds category tags, author meta, a Creative Commons footer, and Disqus comments.
- `_includes/head.html` is a local override of the theme head — this is where all the KaTeX/MathJax/pseudocode/analytics wiring lives.
- `_archive/` is excluded from the build (`exclude:` in `_config.yml`), as are the vendored MathJax asset dirs under `assets/js/`.
- `assets/orbichord/` holds standalone interactive HTML (chord graphs) linked from posts; `assets/data/` holds spreadsheet source data.
- `_site/` is generated output (gitignored).
